# TTS 视频生成方案 — 信号塔Tech 抖音/小红书视频

## 问题

当前的图文卡片方案只适合小红书静态阅读，不适合视频平台：
1. 卡片内容一次性全显示，视频里显得"空白"——观众 2 秒就看完了，剩下 8 秒盯着静态画面
2. Edge TTS 语音机械、不自然，听感差
3. 没有视觉节奏感，缺少动态元素吸引注意力

## 目标

生成 60-90 秒的竖屏视频（1080x1920），适合抖音和小红书视频号，具备：
- 自然流畅的中文配音
- 文字逐段出现的动态效果
- 关键信息高亮强调
- 品牌一致性（信号塔Tech 风格）

## 视频结构（30 秒以内）

```
[0-2s]   封面帧：标题 + hook
[2-26s]  6 张内容卡片，每张 ~4s：
           - 卡片标题 + 正文同时出现
           - 关键词高亮
           - 卡片间无转场，直切
[26-30s] 结尾帧：引导关注
```

## 技术方案

### 1. 语音合成（TTS）

**核心要求：**
- 语速偏快（+15~20%），不拖沓
- 停顿不超过 0.5 秒，保持紧凑节奏
- 30 秒内讲完全部内容

**方案 A：Gemini TTS（推荐）**
- 已有 API key，零额外成本
- 可以在 prompt 里要求快节奏
- 中文质量待测试

**方案 B：Edge TTS + SSML**
- 免费，用 `zh-CN-YunxiNeural`
- SSML 控制：`<prosody rate="+15%">` 加速，`<break time="300ms"/>` 短停顿
- 绝不超过 500ms 停顿

**决策**：先测试 Gemini TTS，不行就 Edge TTS + SSML。

### 2. 视频渲染

**方案 A：Remotion（推荐）**
- React 框架生成视频，代码控制每一帧
- 天然支持文字动画、转场、时间轴
- 输出 MP4，质量高
- 需要 Node.js 环境

**方案 B：FFmpeg + 图片序列**
- 用 Playwright 截图生成帧序列
- FFmpeg 合成视频 + 音频
- 简单但动画效果有限

**方案 C：MoviePy（Python）**
- Python 原生，跟现有代码栈一致
- 支持文字动画、转场
- 性能一般，复杂动画难做

**决策**：用 MoviePy 保持 Python 技术栈一致性，够用就行。如果效果不够好再考虑 Remotion。

### 3. 内容适配

视频版直接复用图文版卡片内容，不单独生成：

| 维度 | 图文版（小红书） | 视频版（抖音） |
|---|---|---|
| 内容来源 | 同一份 | 同一份（共用） |
| 卡片文字 | 100-150 字/张 | 同上，全部显示 |
| 语音脚本 | 无 | 直接朗读卡片内容 |
| 每张时长 | N/A | ~4 秒（语速快） |
| 总时长 | N/A | ≤30 秒 |

### 4. 工作流

```
gen N --video
  ├── 复用已生成的图文卡片内容
  ├── TTS 合成每张卡片语音（快语速，短停顿）
  ├── 计算总时长，确保 ≤30s
  ├── MoviePy 渲染：
  │     ├── 封面帧（2s）
  │     ├── 6 张卡片（直切，语音同步）
  │     └── 结尾帧（2-4s）
  └── 输出 MP4（1080x1920, H.264）
```

## 视觉设计

- 背景：延续现有 Morandi 色系渐变
- 字体：跟卡片一致（STHeiti/Hiragino）
- 文字动画：从左侧淡入滑入，每行间隔 0.3s
- 高亮：关键数字/人名用 accent 色底色标注
- 转场：卡片间用 0.5s 淡入淡出
- 底部常驻：「信号塔Tech」水印 + 页码

## 实施计划

1. **Phase 1**：TTS 测试 — 测试 Gemini TTS vs Edge TTS + SSML，选定方案
2. **Phase 2**：视频渲染原型 — MoviePy 实现基础的"卡片 + 语音"视频
3. **Phase 3**：文字动画 — 逐行淡入、关键词高亮
4. **Phase 4**：集成到 `generate_social.py` — `gen N --video` 命令
5. **Phase 5**：抖音发布流程（手动或半自动）
